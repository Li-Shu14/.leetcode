# CLAUDE.md

## Repository Overview

Personal LeetCode solutions repository written in C++. Contains ~88 solutions covering problems #1 through #1177, organized as individual `.cpp` files. Solutions are authored and submitted via the **VSCode LeetCode Plugin**.

## File Structure

```
.leetcode/
├── 0.skill.cpp              # Algorithm reference/cheat sheet (sorting, searching, data structures)
├── {id}.{problem-name}.cpp   # Individual LeetCode solutions
├── C++与数据结构新的.docx      # C++ and data structures reference document (Chinese)
├── README.md                 # Progress tracker (Chinese)
├── .vscode/settings.json     # VSCode C++ IntelliSense config
└── .gitignore
```

## File Naming Convention

Solutions follow: `{PROBLEM_ID}.{problem-title-kebab-case}.cpp`

Examples: `1.two-sum.cpp`, `200.number-of-islands.cpp`, `34.find-first-and-last-position-of-element-in-sorted-array.cpp`

Special files:
- `0.skill.cpp` — Algorithm reference containing sorting (quicksort, mergesort, heapsort), binary search variants, union-find, KMP, and matrix exponentiation templates
- `00.bookbag_problem.cpp` — Placeholder (empty)

## Code Conventions

### Solution Structure

Each file follows a consistent pattern generated by the LeetCode VSCode plugin:

```cpp
/*
 * @lc app=leetcode id=XXX lang=cpp
 * [XXX] Problem Title
 */

#include <vector>
// other includes as needed
using namespace std;

class Solution {
public:
    // Primary method matching LeetCode's expected signature
    ReturnType methodName(params) {
        return methodName1(params);  // delegates to numbered implementation
    }

    // methodName1() — first approach (author's own)
    // methodName2() — alternative approach (author's own or credited)
    // methodName3() — often marked "Written by others"

private:
    // Helper functions for recursion, DFS, etc.
};
```

### Key Patterns

- **Multiple implementations per file**: Solutions typically contain 2-4 approaches numbered as `method1()`, `method2()`, etc.
- **Attribution**: Later solutions credited with `// Written by others` comments
- **Performance annotations**: Runtime/memory percentiles from LeetCode submissions included as block comments
- **Complexity analysis**: Time and space complexity noted in comments
- **Language**: Code comments mix English and Chinese

### Common Includes

```cpp
#include <vector>
#include <unordered_map>
#include <unordered_set>
#include <queue>
#include <stack>
#include <string>
#include <algorithm>
using namespace std;
```

## Build & Test

- **No build system** — no Makefile, CMakeLists.txt, or scripts
- **No local test framework** — solutions are tested via LeetCode's online judge through the VSCode plugin
- To compile a single solution manually: `g++ -std=c++17 -o solution {file}.cpp`

## Guidelines for AI Assistants

- Preserve the existing file naming convention (`{id}.{problem-name}.cpp`)
- Maintain the multi-solution pattern with numbered methods (`methodName1`, `methodName2`, etc.)
- Include LeetCode metadata header comments (`@lc app=leetcode id=...`)
- Add time/space complexity annotations for any new solutions
- Keep `using namespace std;` consistent with the rest of the codebase
- Comments may be in English or Chinese — match the surrounding context
- Do not add build infrastructure unless explicitly requested
